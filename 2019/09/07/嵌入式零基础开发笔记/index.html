嵌入式学习笔记

HuaGe · 2019-9-7 · 次阅读

第一章，了解Zigbee

1.什么是Zigbee？它能干什么？为什么引入Zigbee？

Zigbee是无限设备之间的一种通信方式，类似人与人之间用普通话进行交流，普通话是一种通信方式。Zigbee通信方式，igbee协议，说的都是一回事。

Zigbee的作用是用来构建无限局域网

各无限通信方式比较

  • 蓝牙：功耗比较低，通信距离短，组建网络节点数少（7~8）
  • WiFi：通信速率比较大，功耗大，等
  • Zigbee：
    • 优点：可以组建大规模网络，功耗低
    • 缺点：通信速率比较小，250kbps
  • 注意：Zigbee只是完成数据的通信的工作，而要实现具体的实用而又强大的功能，需要另外的处理器来进行数据的分析处理

2.IAR软件的安装

  1. 管理员运行IAR kegen PartA.exe,然后点击getID
  2. 运行EW8051-EV-8103-Web.exe,Name随便填

3.创建新的工程

  1. File -> New -> Workspace
  2. Project -> Create New Project -> ok ->指定工程文件夹（可以预先在本地自己创建一个文件夹）和工程文件名
  3. 新建一个文件，保存为.c文件
  4. 把.c文件添加进工程，右击左侧“工程名-Debug”->Add
  5. 编译 -> 点击上方Make ->如果 0 errors就是编译成功

4.选项配置

右击“工程名-Debug” - >options
  1. Genernal Option -> Device 选择 Texas Instruments cc2530F256
  2. Linker -> Output -> 把Override default勾上 ,Allow C-SPY-specific extra output file勾上
  3. Debugger -> Driver -> 选择 Texas Instruments

5.把模块连接上电脑，并安装驱动

在设备管理器中找到smartRF04DD ==> CC2430DB

6.仿真

先要重置仿真器

7.生成hex文件

右键工程->options
Linker -> Extra Output -> 把General extra output file勾上，把overrider default勾上并命名为.hex文件；把output format 改为inter-extended;重新编译后 Debug/Exe文件夹里会生成.hex文件

8.安装烧写软件

  1. Flash image 选中.hex文件
  2. 重置仿真器
  3. 点击perform actions下载
    注意：如果已经在仿真状态，要退出仿真状态才能下载

第二章 硬件电路了解与IO操作

  1. cc2530芯片电路了解

  2. 核心板/电路地板电路了解

  3. 控制IO口的输入输出
    cc2530一共具有21个通用IO口，其中这21个IO口分为三组P0,P1,P2组这些口子都可以作为普通或对应的片上外设（串口，ADC）控制
    P0_0PO_7 P1_0P1_7 P2_0~P2_4
    其中P1_O P1_1具有20mA的输出驱动能力，其余只有4mA

     PxSEL寄存器(x=0,1,2),它的作用是用来配置它对应的IO口是普通IO模式，还是片上外设模式。0表示普通IO，1表示片上外设模式。默认为0   
 注意: P2只有5个IO口，但是P2_1 P2_2是专门用来下载程序用的，所以，这两个IO口不需要配置，因为P2SEL的低3位分别对应的是P2_0  P2_3  P2_4   
 PxDIR寄存器(x=0,1,2),它的作用是用来配置它对应的IO口是输出模式，还是输入模式。0表示输入模式，1表示输出模式。默认为0。具体的对应关系与PxSEL一致。       
 PxINP寄存器(x=0,1,2),它的作用是用来配置它对应的IO口在输入模式下，是上下拉模式，还是高阻态模式。0表示上下拉模式，1表示高阻态模式。   
 注意:由于P2组只有5个引脚，所以该寄存器的高3位用来决定在配置上下拉模式时，到底是上拉还是下拉，0表示上拉，1表示下拉。   
 P2INP   7      6         5      4     3       2        1       0
        P2组  P1组 P0组               P2_4   P2_3    P2_0    

 eg:  P1_2设置为上拉输入
      P1SEL &= 0xFB;//1111 1011把P1_2设置为普通IO模式
      P1DIR &= 0xFB;//1111 1011把P1_2设置为输入模式
      P1INP &= 0xFB;//1111 1011把P1_2设置为上下拉模式
      P2INP &= 0xBF;//1011 1111把P1组设置为上拉模式

    第三章 外部中断

    外部中断配置步骤
    (1)初始化IO口工作在普通IO，上拉输入模式
    (2)使能源中断允许位 PxIEN
    (3)使能组中断允许位 PxIE
    P0IE是IEN1(0xB8)的第5位
    P1IE是IEN2(0x9A)的第4位
    P2IE是IEN2的第1位
    (4)设置上升沿还是下降沿触发(PICTL,0代表上升沿，1代表下降沿)
    (5)使能总中断允许位 EA
    EA是IEN0(0xA8)的第7位
    (6)中断处理函数的写法
